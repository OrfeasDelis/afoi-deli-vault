# Obsidian Usage Rules

## Rule 1 — One note, one purpose

Avoid mixing supplier facts, personal reflections, and order details in the same note.

## Rule 2 — Use templates

Templates make future automation possible.

## Rule 3 — Link aggressively

Every project should link to:

- client
- supplier
- products
- orders
- files
- decisions
- problems

## Rule 4 — Mark uncertainty

Use:

```yaml
confidence: verified | likely | memory_seed | needs_check
```

## Rule 5 — Convert repetition into system

If something happens twice, document it.  
If it happens three times, make a checklist.  
If it happens five times, automate it.

## Rule 6 — Human approval for risk

AI can draft.  
Humans approve external actions.
